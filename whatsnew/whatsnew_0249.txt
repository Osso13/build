9590: Added support for specifying volume attributes in 'floptool flopcreate' [npwoods]
An example command line:
  flopcreate vdk coco_rawdsk_os9_35 newdisk.vdk -name mycooldisk -creation_date "1999-02-28 13:23:47"
Attributes are identified on the command line prefixed with '-'; if this is not the preferred syntax this can be changed.
Implementing this also forced a change to fs::meta_value where the various as_*() calls can now be called without respect to which
  type the fs::meta_value is; this is necessary so that floptool code doesn't need to "own" parsing of the various types of
  fs::meta_value.
And with this change, fs::meta_value::to_string() is now replaced by fs::meta_value::as_string()

10384: Rework QX10 Keyboard implementation [brijohn]
This code reworks the QX10 keyboard code, this fixes several issues with the current implementation.
First it splits the keyboard into both the hasci and ascii variants, previously the mapping was a mixtures of both mappings, it also
  provides a more complete mapping that includes both emulated and natural keyboard support.
I've also added some output_finders for the keyboard LEDS and included a layout for both variants of the keyboard.
Finally the previous version used RS232 for the protocol, however the actual keyboard uses a synchronous clock generated by the
  QX10's timer to drive the keyboard's serial protocol. This was causing issues with keystokes not being processed correctly when
  booting CP/M Plus on the QX10.
